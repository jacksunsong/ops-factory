# 生成完整健康报告(QoS报告)

## 工作原则

本技能专注于汇总已完成的健康分析和根因分析结果，生成完整 QoS 报告。

## 生成步骤

### 步骤1：检查并汇总结果

检查是否同时间段内完成健康分析和根因分析。如完成则直接汇总生成 QoS 报告，无需等待。

**快速检查**：
1. 检查上下文中，用户是否已完成`系统健康初步分析`和`根因分析`
2. 判段两个任务的分析的时间段是否一致
3. 两个报告都存在则立即汇总
**若缺少任一报告**：提示用户需先完成相应分析任务。

### 步骤2：生成 QoS 报告

直接汇总两个报告内容，生成完整报告：

**报告结构**：
```markdown
# QoS完整报告

## 系统健康度概述
[汇总系统健康分析内容]

## 系统根因分析
[汇总根因分析内容]

## 处理建议
[基于分析结果的建议]
```

**要求**：
- 仅汇总，不进行额外分析
- 保持 Markdown 格式
- 报告正文使用中文

## 保存报告
必须将报告保存成markdown文件
- 存储文件名格式：`QoS-report-{系统当前时间}.md`，当前时间使用`yyyyMMddHHmmss`格式，例如：`QoS-report-20260301123000.md`
- 存储路径: `./output`
